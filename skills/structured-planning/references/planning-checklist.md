# Planning Checklist

## Core Questions

- What decision or artifact should exist at the end?
- What files, data, APIs, skills, MCPs, or automations are involved?
- What is already true?
- What must not be changed?
- What would count as proof that the work succeeded?
- What could fail silently?
- What should be reversible?

## Engineering Plan

Use for code and configuration work:

```text
Scope
Files to inspect
Implementation steps
Tests or validation
Git safety
Rollback path
```

## Research Plan

Use for literature, quant, or tool research:

```text
Question
Search strategy
Source quality
Extraction fields
Comparison dimensions
Output format
Open questions
```

## Report Plan

Use for recurring reports:

```text
Audience
Period
Data sources
Required charts/tables
Core conclusion
Risks
Action items
```
