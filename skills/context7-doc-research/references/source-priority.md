# Source Priority

Use this priority order for current technical documentation research.

1. Local project files that pin versions
2. Context7 MCP library docs, when available
3. Official documentation
4. Official release notes or migration guides
5. Official source repository
6. Package metadata
7. Reputable issue discussions only for unresolved bugs or edge cases

Avoid relying on blog posts or forum answers for final API behavior unless official docs are missing and you clearly label the uncertainty.

## OpenAI Exception

For OpenAI API, ChatGPT, Codex, or model guidance:

- Use `openai-docs` first.
- Restrict browsing fallback to official OpenAI domains unless the user asks otherwise.
